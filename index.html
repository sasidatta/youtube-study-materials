2. Adding Files

git add index.html
or add everything:
git add .

This moves files to the staging area for commit.

Files move from working directory to staging area.

3. Committing Changes

git commit -m "Added index.html"

Saves staged changes with a message describing what's changed.

Staged changes are recorded in the project history.

4. Pushing to Remote

git push

Sends committed changes from your local repository to the remote repository (GitHub, GitLab, etc.).

Changes are now visible to collaborators.

5. Pulling Latest Changes

git pull

Fetches and integrates changes from remote to your local repository. Keeps your local code up-to-date.

6. Merging Branches

git merge qa

Merges the qa branch into your current branch (e.g., master).

History of branches gets combined at the merge point.

7. Merge Conflicts

Merge Conflict Example:
You and a collaborator both edited index.html on different branches.
Git can't resolve which changes to keep. The file shows: 
  <<<<<<< HEAD
  <h1 style="background:green;">Welcome!</h1>
  =======
  <h1 style="background:yellow;">Welcome to QA</h1>
  >>>>>>> qa
To resolve: Edit the file, choose the correct lines, remove markers, add & commit again.

8. Pull Requests (PR)

What is a Pull Request?

Proposes merging your branch (e.g., qa) into another branch (e.g., master). Collaboration and code review happen here.

  • Open PR in GitHub/GitLab.
  • Reviewers give feedback, approve/request changes.
  • On approval, PR can be merged into the target branch.
